# 填報後台 建置說明（Google 登入，免密碼）

後台 = 一張 Google 試算表 + Apps Script 網頁應用程式。
學校用**縣網 Google 帳號登入**填報，系統依「權限表」自動判斷角色，
**全程不發放、不保管任何密碼**。

## 一、建立後台（約 10 分鐘，用數辦的 Google 帳號操作）

1. 到 [sheets.new](https://sheets.new) 建立新試算表，命名如「數辦填報系統」。
2. 選單「擴充功能 → Apps Script」。
3. 把 `backend/Code.gs` 的內容全部貼進 `Code.gs`（取代原內容）。
4. 左側「檔案 ＋ → HTML」，命名 **Index**（大小寫需一致），貼上 `backend/Index.html` 的內容。
5. 儲存（Ctrl+S）。

## 二、部署（資安關鍵設定）

1. 右上「部署 → 新增部署 → 類型選『網頁應用程式』」。
2. **執行身分：我**（資料寫入數辦的試算表，學校看不到整份表）。
3. **具有存取權的使用者：「網域內所有人」**（僅限縣網 Google 帳號；
   若各校帳號不在同一個 Workspace 網域，改選「任何擁有 Google 帳號的使用者」，
   系統仍會逐一比對權限表 Email，未授權帳號一律擋下）。
4. 按「部署」→ 授權（第一次會要求核准試算表/Gmail/雲端硬碟權限）→ 複製 `/exec` 網址。
5. 開啟 `/exec` 網址測試：未加入權限表的帳號應看到「尚未授權」畫面（這是正確的）。

> 之後修改 Code.gs 要生效，須「部署 → 管理部署 → 編輯 → 新版本」。

## 三、填權限表

1. 開啟 `backend/權限表範本.csv`（Excel 可直接開，198 校已列好）。
2. 在 Email 欄填入各校填報人的縣網 Google 帳號；最後三列填數辦夥伴、承辦人、管理員。
3. 全選複製，貼到試算表的「權限表」工作表（第一次執行後系統會自動建立各工作表；
   或自行建立名為「權限表」的工作表再貼上）。
4. 角色四級：`學校`（填報＋看本校）、`數辦`（看全縣彙整）、
   `承辦`（彙整＋寄提醒信＋產生前台資料檔）、`管理`（全部功能）。
   一人可多列（例如承辦人同時是某校填報人）。

## 四、接上官網

1. 打開官網 repo 的 `data/config.js`，把 `portalUrl` 填入 `/exec` 網址。
2. git push 後，官網右上角與「公開觀課」「減授課」頁會自動出現「填報系統」按鈕。

## 五、彙整回寫官網（承辦人操作）

1. 後台「全縣彙整」分頁 → 各校填報後自動更新；
   「第一期領據／第二期領據／收支結報表」三欄請直接在試算表「彙整」工作表改成「已繳」。
2. 按「產生前台資料檔」→ 複製產出的內容 → 貼到官網 repo `data/relief-status.js`
   → `data/config.js` 的 `reliefDemo` 改 `false` → git push，公開網站即顯示真實彙整。
   （也可以把產出的內容貼給 Claude，請它幫你更新並推送。）

## 六、資安檢核表

- ✅ 不自管密碼：登入走 Google 帳號（縣網 SSO），忘記密碼由 Google/縣網處理。
- ✅ 最小權限：學校帳號只能填報與查看自己學校；彙整僅數辦以上可見。
- ✅ 資料分級：金額明細只存在數辦的試算表；公開網站僅放彙整後統計。
- ✅ 全程 HTTPS（script.google.com 與 github.io 皆強制加密）。
- ✅ 稽核軌跡：每筆填報都有時間戳與填報人 Email，試算表另有版本紀錄。
- ⚠️ 試算表本身**不要**用「知道連結的人皆可檢視」分享；只給數辦成員編輯權。
- ⚠️ 提醒信使用 Gmail 配額（教育帳號每日約 1,500 封），198 校綽綽有餘。
